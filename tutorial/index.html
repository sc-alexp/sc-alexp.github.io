Tutorial

Overview

In a time long long ago in a room not far away, in a time when dinosaurs roamed the earth, sc_alexp learned from sc_nickv that ServiceNow is able to parse HTML in the Additonal comments/Work notes fields!

It is not able to do it with no help, however! There is a trick!

In order for ServiceNow to parse the html in a message, it must be surrounded by the proper tags.


HTML that will not be rendered in Additonal comments:

<a href="https://iastate.edu/">Iowa State University</a>

HTML that will be rendered in Additonal comments:

[code]<a href="https://iastate.edu/">Iowa State University</a>[/code]


"Well that's cool and all, but I don't know HTML"

I have good news for you! That is what this tool is for!!



Formatting your response for a ticket

At the main page of this site, you will find the editor. You can change how it arranged using the and buttons at the bottom. Either way your screen will be split into two halves.

The Preview

You will never interact with the preview, but it is a good indicator of what your response will look like to the customer. It updates in real time as you edit your response.

Given that this is a different website with different code than ServiceNow, it may not be 100% accurate in every case but I have found that it is very reliable. It may also change over time as ServiceNow updates but use your own discretion.

The Editor

The star of the show. The editor in this tool is basically a fancy custom markdown parser. The buttons will apply markdown syntax (whenever possible) and you will see the rendered result in the preview.

Never copy or paste directly from the editor or the preview. This will result in your message not being rendered correctly and being confusing for the caller.

When you are ready to copy your message into a ticket, always use the green Copy for Ticket button. This will properly format your response so that ServiceNow renders it to the caller.

Buttons

For most buttons with the text you want to alter, you first select the text and then click the button to alter it how you want, the same as you would in any normal text editor. I recommend you experiment, try adding a link, try bolding text.


Templates

To better standardize and increase quickness in response times, there are a few basic templates that define basic formats for replies and work notes. You can find these by clicking Templates in the navbar at the top of the page.

Find the template you want, and click Use this Template. This will take you back to the editor with the template autofilled and ready for you to edit!

If you refresh, it will reapply the template but remove your changes. To get rid of the template, simply click the button to clear the text field and current set template.

If you have any template ideas, let me know! Note these templates will never have any information of substance in them, and they are only for basic formatting of responses.
